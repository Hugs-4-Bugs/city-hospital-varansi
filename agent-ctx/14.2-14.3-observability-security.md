## Phase 14.2: Observability (6 files)

1. **MetricsCollector** (`src/lib/observability/metrics-collector.ts`)
   - Singleton class with Prometheus-compatible metrics
   - Counters: api_requests_total, credits_consumed_total, workflow_executions_total, ai_requests_total, ai_cost_total, payment_failures_total, payment_success_total, anomaly_alerts_total, competitor_scans_total
   - Histograms: api_request_duration_seconds, workflow_duration_seconds, db_query_duration_seconds, redis_operation_duration_seconds
   - Gauges: active_users, credits_remaining, queue_depth, websocket_connections
   - Methods: incrementCounter(), observeHistogram(), setGauge(), getMetrics(), toPrometheusFormat()

2. **Prometheus Metrics Endpoint** (`src/app/api/metrics/route.ts`)
   - GET /api/metrics returns Prometheus text format
   - Collects real data from database (user counts, credit balances, payment stats, AI costs, workflow counts)
   - Measures DB query latency

3. **Detailed Health Check** (`src/app/api/health/detailed/route.ts`)
   - GET /api/health/detailed returns component-level health
   - Checks: Database (query latency), Memory (heap usage), Process (uptime, CPU)
   - Status: healthy / degraded / unhealthy
   - Returns version, environment, metrics snapshot

4. **Sentry Integration** (`src/lib/observability/sentry.ts`)
   - initSentry() with NEXT_PUBLIC_SENTRY_DSN env var
   - captureException(), captureMessage() helpers
   - setSentryUser() for user context enrichment
   - addBreadcrumb(), clearBreadcrumbs() for breadcrumb management
   - No-op fallback when DSN is not configured
   - withSentryContext() for scoped capture

5. **Grafana Dashboard** (`deploy/grafana/dashboards/acquisitionos-overview.json`)
   - Pre-built dashboard with panels for: API Rate & Latency, Error Rate, Credit Usage, AI Cost, Payment Success/Failure, Workflow Metrics, Realtime Connections, DB Performance, Queue Depth, Anomaly Alerts

6. **Prometheus/Grafana Deploy Configs**
   - `deploy/grafana/provisioning/datasources/prometheus.yml` — Auto-provision Prometheus datasource
   - `deploy/grafana/provisioning/dashboards/dashboard.yml` — Auto-provision dashboard
   - `deploy/prometheus/prometheus.yml` — Scrape config for localhost:3000/api/metrics
   - `deploy/prometheus/alerts.yml` — 7 alert rules: HighErrorRate, HighLatency, CreditExhaustion, PaymentFailureSpike, WorkflowFailureSpike, DBLatency, HighMemoryUsage

## Phase 14.3: Security Hardening (10 files)

1. **Rate Limiter** (`src/lib/security/rate-limiter.ts`)
   - Sliding window algorithm with in-memory store
   - Pre-configured limiters: API (100/min/user), Auth (5/min/IP), AI (20/min/user), Webhook (1000/min/IP), Upload (10/min/user), Export (5/min/user)
   - withRateLimit() middleware returning 429 with Retry-After
   - Rate limit headers: X-RateLimit-Limit, Remaining, Reset

2. **CORS Config** (`src/lib/security/cors-config.ts`)
   - Strict CORS from ALLOWED_ORIGINS env var
   - Blocks null origins in production
   - Wildcard subdomain support
   - withCors() and withCorsHandler() middleware

3. **CSRF Protection** (`src/lib/security/csrf-protection.ts`)
   - Double-submit cookie pattern with crypto.randomBytes
   - Origin/Referer validation
   - Exempt paths: /api/webhooks/*, /api/health/*
   - withCsrfProtection() middleware
   - Bearer token and X-Requested-With bypass for API clients

4. **Input Validator** (`src/lib/security/input-validator.ts`)
   - SQL injection detection (14 patterns)
   - XSS sanitization (12 patterns)
   - Path traversal detection (9 patterns)
   - Command injection detection (9 patterns)
   - File upload validation with magic bytes
   - withValidation() Zod middleware

5. **Security Headers** (`src/lib/security/security-headers.ts`)
   - CSP, X-Frame-Options DENY, X-Content-Type-Options nosniff
   - Referrer-Policy, Permissions-Policy, HSTS
   - Cross-origin policies (COOP, COEP, CORP)
   - X-Request-ID with UUID generation
   - withSecurityHeaders() middleware

6. **Webhook Security** (`src/lib/security/webhook-security.ts`)
   - HMAC signature verification for Stripe, Razorpay, Telegram, Meta, Twilio
   - Constant-time comparison (crypto.timingSafeEqual)
   - Timestamp tolerance (5 min for Stripe)
   - withWebhookVerification() middleware

7. **Upload Security** (`src/lib/security/upload-security.ts`)
   - File type whitelist (image, document, data categories)
   - Size limits (10MB default)
   - Magic bytes validation
   - Filename sanitization
   - Safe storage path generation with UUID directories
   - Double extension detection
   - withUploadSecurity() middleware

8. **JWT Security** (`src/lib/security/jwt-security.ts`)
   - Algorithm whitelist (HS256 only)
   - Token blacklisting with JTI support
   - Refresh token family/replay detection
   - Suspicious activity tracking per user
   - OAuth state validation
   - PKCE support (S256 method)
   - generateSecureAccessToken() with JTI

9. **Security Audit** (`src/lib/security/security-audit.ts`)
   - 36 event types with severity levels (critical/high/medium/low/info)
   - Failed auth tracking with brute force detection
   - Privilege escalation detection (role hierarchy)
   - withSecurityAudit() middleware
   - runSecurityAudit() automated report generation
   - Database persistence via AuditLog model

10. **Security Scan Script** (`scripts/security-scan.ts`)
    - Dependency vulnerability scanner (bun audit)
    - Environment variable security checks
    - Sensitive file detection
    - .gitignore validation
    - Hardcoded secret pattern scanning
    - JSON report generation (security-report.json)
